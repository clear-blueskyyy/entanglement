# 纠缠 — 产品需求与技术交付文档

> 本文档面向开发执行方（AI编程助手/开发者），描述产品目标、功能规格、交互行为和约束条件。技术方案和实现细节由执行方自主决策，本文档仅定义"做什么"和"做到什么程度"，不规定"怎么做"。

---

## 1. 产品定义

**产品名称**：纠缠（Entanglement）

**一句话描述**：输入两个看似无关的词，AI推理出它们之间出人意料的关联路径，并以交互式可视化呈现。

**核心体验关键词**：意外感、探索感、轻量

**不是什么**：
- 不是知识图谱工具——不需要全面，需要有趣
- 不是百科查询——不追求准确性覆盖，追求关联的巧妙
- 不是生产力工具——是好奇心玩具

---

## 2. 用户场景

**场景A - 每日一玩**
用户打开页面，首页展示今日自动生成的一组配对（如"企鹅"和"加密货币"）。用户点击"发现关联"，路径逐步展开，感受到"居然还能这么连"的惊喜。

**场景B - 自由探索**
用户自己输入两个词（如"雨林"和"说唱"），点击"纠缠"，看到AI找到的关联路径。如果觉得不够妙，可以换一条路径或换一个词重新生成。

**场景C - 分享传播**
用户看到一条特别妙的路径，点击生成分享卡片，截图发给朋友。

---

## 3. 功能规格

### 3.1 核心功能：关联路径生成

**输入**：两个词或短语（中英文均可）

**输出**：1-2条关联路径，每条包含：
- 3-5个中间概念节点
- 相邻节点之间的一句话关联解释
- 路径整体的一句话总结（点出最妙的关联）

**质量要求**（这是产品成败的关键）：
- 每一步关联必须具体、可理解，禁止出现"都反映了时代发展""都是全球化的产物"这类空话
- 禁止使用万能中介概念（如"技术进步""全球化""人类文明"等过于宽泛的概念充当桥梁节点）
- 路径中至少有一跳是多数人想不到的——这是产品的价值所在
- 整条路径读下来应该像一个有趣的思维故事，而不是学术论文的参考文献链

**交互行为**：
- 路径默认不全展开，先展示首尾两个词和中间节点数量，营造悬念
- 用户主动点击"展开下一跳"逐个揭示中间节点
- 完全展开后显示路径总结
- 提供"换一条路径"按钮（同一对词可能有不同路径）

### 3.2 每日自动配对

**行为**：首页自动展示一组当日配对，用户无需输入即可体验

**配对规则**：
- 两个词来自不同领域（如科技+文化、自然+政治），同领域的配对太容易猜到关联
- 配对要有一定话题性，避免"苹果"和"椅子"这种完全随机的无聊组合
- 配对词必须是真实存在的事物/概念/事件，禁止 AI 编造热点或不存在的新闻
- 每次打开首页随机生成配对（不要求同一天固定），用户可点击"换一组"主动刷新

**交互行为**：
- 首页展示配对卡片：两个词 + 中间一个"?"标识 + "发现关联"按钮
- 提供"换一组"按钮刷新配对

### 3.3 自由探索

**行为**：用户手动输入两个词进行纠缠

**交互行为**：
- 两个输入框，placeholder引导（如"随便写点什么""再写个不一样的"）
- 输入后点击"纠缠"按钮生成路径
- 支持从历史配对中"换一边"——保留一个词，重新输入另一个

### 3.4 历史记录

**行为**：记录用户探索过的纠缠，可回看

**要求**：
- 存储在浏览器本地，不需要注册登录
- 按时间倒序展示
- 每条记录显示：两个词 + 路径总结 + 日期
- 点击可重新展开完整路径

### 3.5 分享卡片

**行为**：生成一张视觉好看的图片，包含配对词和路径总结

**要求**：
- 可截图下载
- 卡片上有"纠缠"品牌标识
- 视觉风格与整体产品一致

## 5. 视觉风格

**整体方向**：暗色系、克制、有质感。不是花哨的科技风，是安静的探索感。

**关键词**：深色背景、微光、渐变、留白

**具体指引**：
- 背景：深色（接近黑色但不纯黑，带一点深蓝或深灰调）
- 节点：有柔和的微光/光晕效果，像暗夜里的信号灯
- 连线：细线，带一点渐变或发光
- 字体：无衬线，中英文都好看
- 动效：克制但有——节点展开时有入场感，不是突然蹦出来
- 整体氛围参考：天文类App的夜间模式、深空主题的可视化作品

**自由度**：具体配色、字体选择、动效参数由执行方决定，以上是方向指引而非像素级spec。

## 6. 技术约束

### 6.1 硬性约束

- **AI推理能力**：必须接入大语言模型API进行路径推理，不能用硬编码的关联数据
- **API Key管理**：Key不能硬编码在代码中，必须使用环境变量；生产环境Key仅存放在部署平台的服务端环境变量，不能下发到浏览器
- **中文支持**：产品面向中文用户，所有文案和默认内容为中文，但也要兼容英文输入
- **响应式**：至少在桌面端和手机端都能正常使用，不要求完美适配但不能布局崩坏
- **部署架构**：MVP不需要传统后端服务器，但AI调用必须经过Serverless API代理，禁止浏览器直连模型API
- **免费运行**：选用的API和服务必须有免费额度，个人使用不应产生持续费用
- **可分享访问**：部署后需提供可公开访问的HTTPS链接，用户可直接通过网址打开和体验

### 6.2 AI模型接入方案（重点）

本项目需要两个AI能力：路径推理 + 热点配对生成。以下为推荐的接入方案，执行方可根据实际情况选择或组合：

**方案A：智谱GLM API（推荐主力）**
- 模型：GLM-4-Flash
- 免费情况：**永久免费**，非限时赠送
- 上下文：128K
- 注册地址：https://open.bigmodel.cn
- 优点：免费且无过期压力，国内服务器延迟低，高峰期稳定
- API端点：`https://open.bigmodel.cn/api/paas/v4/chat/completions`
- 认证方式：JWT Token（注册后在API Key管理中获取，需按文档生成鉴权Token）
- 注意：智谱的鉴权方式与OpenAI不同，不是直接传API Key，而是需要用API Key生成JWT Token。执行方需查阅智谱官方文档实现鉴权

**方案B：扣子工作流（备选/降级）**
- 在扣子平台（coze.cn）创建一个工作流：接收两个词 → 调用大模型推理 → 返回结构化结果
- 前端通过Serverless API代理调用该工作流
- 优点：不单独计费Token，扣子账号即可使用；工作流可视化调试，prompt优化直观
- 缺点：增加外部依赖；扣子API有频率限制
- 适用场景：作为智谱API不可用时的降级通道

**推荐策略**：主力用方案A（智谱GLM-4-Flash），代码层面做好抽象，AI调用统一走Serverless网关接口，方便在A/B之间切换。当智谱API响应超时或出错时，网关自动降级到方案B（扣子工作流）。

### 6.3 其他推荐但不强制

- 前端框架：React + Vite + TypeScript
- 路径可视化：React Flow 或任何适合节点+连线交互的库
- 样式：TailwindCSS
- 部署：Vercel（免费层，支持公开URL与自定义域名）

**如果执行方有更好的技术选择，可以替换，但需要说明理由。**

### 6.4 明确不做

- 用户注册/登录系统
- 后端数据库
- 多人协作/社交功能
- 付费/商业化功能
- 移动端原生App

### 6.5 部署与外部访问规范

- 部署平台采用静态托管 + Serverless函数（推荐Vercel）
- 上线后必须提供公开HTTPS访问地址，可直接分享给他人体验
- 支持可选绑定自定义域名，便于传播与品牌识别
- 桌面端与手机端主流浏览器可打开，不出现布局崩坏和白屏
- 对外错误提示需友好，API失败时展示可理解的重试引导

### 6.6 术语约定（统一口径）

- **客户端**：指浏览器中的前端单页应用（SPA）
- **Serverless网关**：指部署平台上的无状态函数接口层（如Vercel Functions），负责鉴权、限流、调用上游模型
- **上游模型服务**：指智谱GLM或扣子工作流等第三方推理服务
- **禁止直连**：客户端不得直接请求上游模型服务，所有AI请求必须先到Serverless网关
- **统一错误结构**：网关对外返回一致的错误JSON，前端仅按统一结构处理

## 7. 数据与接口契约

### 7.1 数据结构参考

以下定义了核心数据的结构和字段含义，供实现参考。具体命名和扩展由执行方决定。

```
一次纠缠结果：
- id: 唯一标识
- termA: 起始词
- termB: 目标词  
- paths: 路径数组（1-2条）
  - 每条路径包含：
    - nodes: 节点数组（3-5个中间节点）
      - 每个节点包含：term(概念名)、connectionToNext(到下一跳的关联)、detail(补充说明)
    - summary: 路径总结
- surpriseIndex: 意外指数 1-10（可选，AI评估）
- createdAt: 创建时间
```

### 7.2 最小接口契约（MVP）

为保证前后端与Serverless网关协作稳定，MVP阶段至少实现以下接口：

**1）生成关联路径**
- 路径：`POST /api/entangle`
- 请求体：
```json
{
  "termA": "深海采矿",
  "termB": "韩国大选",
  "regenerate": false
}
```
- 成功响应（200）：
```json
{
  "requestId": "req_xxx",
  "data": {
    "id": "ent_xxx",
    "termA": "深海采矿",
    "termB": "韩国大选",
    "paths": [
      {
        "nodes": [
          {
            "term": "稀土供应链",
            "connectionToNext": "...",
            "detail": "..."
          }
        ],
        "summary": "..."
      }
    ],
    "createdAt": "2026-05-05T12:00:00.000Z"
  }
}
```

**2）获取今日配对**
- 路径：`POST /api/daily-pair`
- 请求体（可空）：
```json
{
  "date": "2026-05-05"
}
```
- 成功响应（200）：
```json
{
  "requestId": "req_xxx",
  "data": {
    "date": "2026-05-05",
    "termA": "深海采矿",
    "domainA": "自然/科学",
    "termB": "韩国大选",
    "domainB": "政治/社会"
  }
}
```

**3）统一错误响应**
- 失败响应（4xx/5xx）：
```json
{
  "requestId": "req_xxx",
  "error": {
    "code": "E_MODEL_TIMEOUT",
    "message": "模型响应超时，请稍后重试",
    "retryable": true
  }
}
```

**错误码建议**：
- `E_BAD_REQUEST`：参数不合法
- `E_RATE_LIMIT`：触发限流
- `E_MODEL_TIMEOUT`：上游模型超时
- `E_PROVIDER_UNAVAILABLE`：上游服务不可用
- `E_INVALID_MODEL_OUTPUT`：模型返回非预期JSON
- `E_INTERNAL`：网关内部错误

## 8. AI Prompt 要求

路径推理的质量直接决定产品体验。以下为prompt的核心要求，具体措辞由执行方编写和优化：

**必须包含的指令**：
- 路径由3-5个中间节点组成
- 每步关联必须具体，给出明确的因果/类比/历史/结构关系
- 禁止空泛关联（如"都反映了XX趋势"）
- 禁止万能中介节点（如"技术进步""全球化""社会变迁"等过于宽泛的概念）
- 路径中至少一跳应当是大多数人想不到的
- 输出必须为严格JSON格式
- 每条路径需要一句话总结最妙的关联

**返回格式要求**：严格JSON，结构对应上方数据结构

**配对词生成的prompt要求**：
- 生成5个不同领域的话题，每次随机选取2个不同领域进行配对
- 领域至少覆盖：科技、政治/社会、文化/艺术、自然/科学、商业/经济
- **话题必须是真实存在、可被验证的事物/概念/事件，严禁 AI 编造不存在的内容**
- 话题之间领域差距越大越好，避免同领域出现
- 返回JSON格式包含话题名和领域

## 9. 开发阶段与交付标准

按以下顺序推进，每个阶段独立可验证：

### 阶段0：项目骨架与可访问部署

**交付物**：前端基础页面 + Serverless API代理骨架，生成一个可公开访问的测试链接

**验证方式**：在非开发者本机（如手机4G网络）打开链接，可正常访问首页

**验收标准**：
- HTTPS链接可直接访问
- AI请求经Serverless代理转发
- 浏览器端代码与网络请求中不出现模型供应商Key

### 阶段1：路径推理跑通

**交付物**：一个页面，两个输入框，点击按钮后通过代理接口获取并展示AI返回的关联路径（纯文本即可）

**验证方式**：输入"深海采矿"和"韩国大选"，页面展示出至少一条有3个中间节点的结构化路径

**验收标准**：路径中不出现"全球化""技术进步"等万能中介，每步关联具体可理解

### 阶段2：路径可视化

**交付物**：路径以交互式节点+连线的方式展示，支持逐步展开

**验证方式**：路径不再是一坨文字，而是有视觉节奏的探索体验

**验收标准**：
- 初始只显示首尾节点，中间节点数量提示
- 点击可逐个展开中间节点
- 展开有入场动效（不能是突然出现）
- 整体暗色风格（深色背景，非纯黑）
- AI 推理等待期间有动态文案（不能白屏或静态 spinner）
- AI 失败时展示友好错误提示，提供重试入口（不能无提示白屏）

### 阶段3：每日配对+首页

**交付物**：打开页面自动展示今日配对，有首页体验

**验证方式**：无需输入任何内容即可开始体验

**验收标准**：配对来自不同领域，点"发现关联"可直接进入路径展开

### 阶段4：完整体验

**交付物**：加入历史记录、分享卡片、动效打磨

**验证方式**：完整走通"打开→浏览→探索→分享"流程

**验收标准**：
- 历史记录可正常写入、展示、点击回看
- 分享卡片包含两个词 + 路径总结 + "纠缠"品牌标识，可截图或下载
- 完整流程（打开→配对→探索→展开→分享）无阻断、无白屏
- 在手机端（非开发者设备）可正常体验全流程

### 阶段验收清单（可勾选）

- [ ] **阶段0通过**：公开HTTPS链接在桌面和手机网络均可访问，且前端请求中无供应商Key
- [ ] **阶段1通过**：输入任意两词可返回结构化路径，含3-5个中间节点且关联具体
- [ ] **阶段2通过**：路径可逐步展开，首尾与中间节点层次清晰，具备基本入场动效
- [ ] **阶段3通过**：首页可直接体验今日配对，配对词来自不同领域
- [ ] **阶段4通过**：历史记录、分享卡片可用，手机端完整流程无阻断

## 10. 已知风险提示

以下是我预见到的技术风险，供执行方提前考虑：

| 风险 | 说明 | 期望的处理方式 |
|------|------|----------------|
| AI返回非JSON | 大模型偶尔不听话，返回带说明文字的JSON | 需要有容错解析：提取JSON块 → 解析 → 失败时友好提示用户 |
| 路径质量不稳定 | 有时很妙有时很无聊 | 提供"换一条路径"功能；prompt持续优化 |
| API调用失败 | 网络问题、额度用尽或模型服务异常 | 主模型超时自动降级到备选API；页面需要有错误状态展示，不能白屏或报错无提示 |
| API Key泄露与滥用 | 浏览器直连模型API会暴露凭证，公开分享后可能被刷额度 | 必须通过Serverless代理转发，服务端环境变量存Key，并加基础限流与错误熔断 |
| 首次加载慢 | 路径推理需要等待AI响应 | 需要有加载状态，最好有有意义的等待体验（如"正在思考两个陌生人的关系……"） |
| 智谱鉴权复杂 | 智谱API使用JWT Token鉴权，与OpenAI格式不同 | 执行方需查阅智谱官方文档正确实现鉴权；鉴权失败时可降级到扣子工作流 |
