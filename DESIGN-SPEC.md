# ENTANGLEMENT / 纠缠 —— 产品UI/UX设计规范

> **最后更新**: 2026-05-05
> **设计阶段**: MVP 视觉交互定义已定稿
> **设计工具**: Figma 原型 / HTML+CSS+React 实现
> **设计对标**: Nautilus 杂志（克制、学术感、大留白）

## 一、产品定位与设计目标

### 1.1 产品定位

| 维度 | 规格 |
|------|------|
| **名称** | ENTANGLEMENT / 纠缠 |
| **标语** | 万物皆有暗线 |
| **体验模式** | 碎片时间的智性消遣 |
| **使用场景** | 2-5分钟单次体验闭环，不需要用户留存 |
| **视觉调性** | 暗色、克制、有质感（像深夜的编辑室） |

### 1.2 设计核心价值

- **尊重用户的思维过程**：不是一次性摊平所有信息，而是逐步揭示，让用户有时间消化每一步
- **仪式感的路径体验**：从"输入 → 等待 → 揭示 → 收尾"全程都有节奏感
- **克制但不冷漠**：琥珀金点缀色不是为了炫耀，而是在深黑中建立一条温暖的视觉引导线

## 二、视觉设计基线

### 2.1 色彩系统

| 语义 | 色值 | 用途 |
|------|------|------|
| **背景深** | `#07080f` | 页面底色 |
| **背景中** | `#0b1026` | 环境渐变、深度 |
| **卡片底** | `rgba(10,16,28,0.9)` | 内容卡片背景 |
| **点缀色主** | `#d4a853`（琥珀金） | 按钮、标题、高亮连线 |
| **点缀色副** | `#c49a3c` | 点缀色渐变、hover 状态 |
| **文本主** | `#f0ece6`（冷白） | 标题、重要正文 |
| **文本次** | `#b0ada6`（银灰） | 说明文字、副本 |
| **文本弱** | `#7a7770`（灰褐） | 禁用、占位符 |

**渐变配方**:
```css
background: radial-gradient(ellipse at 18% 15%, rgba(212,168,83,0.06), transparent 40%),
            linear-gradient(180deg, #07080f 0%, #0b1026 45%, #07080f 100%);
```

### 2.2 字体系统

| 层级 | 字体 | 用途 |
|------|------|------|
| **品牌英文** | Playfair Display | "ENTANGLEMENT" 品牌名 |
| **中文标题** | Noto Serif SC | "纠缠"、路线标题、节点名 |
| **正文** | Inter / 系统无衬线 | 所有副本、说明文字 |

**字号定义**:
- 品牌名英文: `13px`, 字间距 `0.35em`
- 品牌名中文: `clamp(2.6rem, 6vw, 4.2rem)`, 字重 700
- 路线标题: `1.35rem`, 字重 700
- 一级标题: `clamp(1.3rem, 2.4vw, 1.8rem)`
- 二级标题: `14px`, 字重 600
- 正文: `13-14px`, 行高 `1.75-1.8`

### 2.3 间距与圆角系统

```
--radius-lg: 28px      // 卡片、Hero 区
--radius-md: 20px      // 路径舞台、内容卡
--radius-sm: 14px      // 路径标签、小卡片
--radius-pill: 999px   // 按钮、药片状标签

间距规律: 8px / 12px / 16px / 20px / 24px / 28px (8px 基线递增)
```

## 三、页面结构与交互

### 3.1 首屏 Hero 布局

```
┌─────────────────────────────────────────────────┐
│  星网背景（Canvas SVG）↙ ↘ 环境光晕（blur）   │
│                                                 │
│   ┌────────────────┐    ┌─────────────────┐    │
│   │  ENTANGLEMENT  │    │  PAIR BUILDER   │    │
│   │    纠缠        │    │                 │    │
│   │ 万物皆有暗线   │    │  [输入A] ↔ [输入B] │
│   │                │    │                 │    │
│   │ [说明文案...]  │    │  [开始纠缠]     │    │
│   │                │    │                 │    │
│   │ ● 逐跳揭示   │    │  [今日推荐配对]   │    │
│   │ ● 像杂志...  │    │  [示例1] [示例2]  │    │
│   │ ● 克制质感   │    │                 │    │
│   └────────────────┘    └─────────────────┘    │
│                                                 │
└─────────────────────────────────────────────────┘
```

**交互规则**:
- 首屏高度占满 viewport，左右两列在 900px+ 宽度时双列，以下单列
- 星网背景各天体独立缓动（9–14s），星线连接动画 42s ease-in-out infinite
- 所有输入框focus时边框变琥珀金，加发光阴影
- 用户输入后，`[开始纠缠]` 按钮可用

### 3.2 结果页 Hero 降级处理

当用户看到结果后，首屏应该自动**"收缩上移"**：

```css
.app.has-result .card-hero {
  transform: scale(0.96) translateY(-10px);
  opacity: 0.62;
  pointer-events: auto;
  gap: 14px;
  padding: 22px;
}
```

用户可以：
- 滚动回顶修改输入词
- 点击"换一条路径"重新生成
- 结果页下方有"换一对试试"按钮可重置（该按钮仅在收尾仪式完成后（所有节点揭示完毕）出现，非始终可见。）

### 3.3 Constellation 星座轨道（核心体验）

```
深海采矿 ─────── ? ─────── ? ─────── ? ─────── 韩国大选
  ●        ↙ ↖      ↙ ↖      ↙ ↖      ↙ ↖      ●

├─────────────────────────────────────────────────┤
  当点击"揭示下一跳"时：
  ┌─ 粒子流动 (400ms)
  │  ? → 消融，上方节点名渐现
  │  粒子从起点沿连线流向该节点 (amber color, 带尾迹)
  ├─ 侧栏同时显示 detail
  └─ 按钮文案更新为"揭示下一跳（1/4）"
```

> 实际实现为**纵向**布局（`display: grid`，节点从上到下排列，连线为竖向 `edge-line`）

**节点样式**:

| 类型 | 边框色 | 背景 | 光晕 | 交互 |
|------|--------|------|------|------|
| start/end | 琥珀金 | 带渐变 | 24px shadow | 不可点击 |
| middle (隐藏) | 琥珀金半透明 dashed（`rgba(212, 168, 83, 0.18)`，即 `var(--border-mid)`） | 半透明 | 无 | 不可点击 |
| middle (已揭示) | 淡银 | 深色半透 | 无 | 可点击展开详情 |
| middle (ceremony) | 琥珀金 | 带渐变 | 28px glow | 固定发光 |

**连接线动效**:
- 初始: `linear-gradient(180deg, rgba(255,255,255,0.14), rgba(255,255,255,0.02))` 淡灰
- 已激活: `linear-gradient(180deg, rgba(244,223,155,0.95), rgba(212,168,83,0.18))` 琥珀渐变 + glow
- ceremony: 始终琥珀金亮起，加脉冲

## 四、交互流与仪式感设计

### 4.1 完整交互序列

```
[首屏] 
  ↓ 输入 A、B（或点击示例）
  ↓ 点击 [开始纠缠]
[等待态]
  ↓ 加载动画 + loading hint 轮换
  ↓ 通常 10-60 秒（取决于模型响应速度，前端超时上限 60s）
[结果展开]
  ├─ 首屏缩小上移
  ├─ Constellation 轨道出现
  ├─ 路线元信息卡展开
  ├─ 星座轨道中起点/终点亮起
  └─ 中间节点全为 ?
      ↓ 用户点击 [揭示下一跳]
    ┌─ 粒子流从起点→第1个?
    ├─ 500ms 粒子到达，? 消融
    ├─ 第1个节点出现 + 侧栏detail展开
    ├─ 侧栏可点击节点看详情
    └─ 循环至所有节点揭示完成
      ↓
[收尾仪式 Ceremony]
    ├─ 所有节点同时发光脉冲 (1.2s)
    ├─ 连线全亮 + 光脉冲从起点→终点 (1.4s)
    ├─ 路径概览卡淡入 (0.6s, 延迟0.3s)
    ├─ 底部引导按钮出现
    └─ "再看一条路" / "换一对试试"
```

**揭示下一跳按钮状态变化**：当所有中间节点揭示完毕但终点连线未点亮时，按钮变为"点亮终点连线"；终点连线点亮后变为"整条路径已经点亮"。

### 4.2 动效参数

| 动效 | 时长 | 缓动 | 触发条件 |
|------|------|------|---------|
| 粒子流落 | 420ms | cubic-bezier(0.4,0,0.2,1) | 点击揭示 |
| 节点出现 | 400ms | cubic-bezier(0.4,0,0.2,1) | 粒子到达后 |
| 侧栏卡片 | 400ms | 同上 | 节点出现同时 |
| ceremony 脉冲 | 1200ms | 同上 | 全部揭示完 |
| 光脉冲流 | 1400ms | 同上 | ceremony 中 |
| 路径概览淡入 | 600ms | 同上 | ceremony 后 延迟300ms |

**原则**: 所有动效都应该给用户"思考的时间"，而不是"等待的焦虑"。

### 4.3 空态体验

当还没有结果时，下方显示"How It Feels"区：

```

│ HOW IT FEELS                    │
│                                 │
│ 它不是答案页，而是一场被慢慢   │
│ 揭开的路径体验。                │
│                                 │
│ ┌──────────┐ ┌──────────┐ ┌──┐│
│ │ 01       │ │ 02       │ │03││
│ │ 抛出一对 │ │ 一跳一跳 │ │最│
│ │ 词       │ │ 揭示     │ │后│
│ │[说明文]  │ │[说明文]  │ │才│
│ └──────────┘ └──────────┘ │亮│
```

## 七、设计细节与品质保证

### 7.1 光学调整

- 琥珀金亮度：在深黑背景上应该能自然吸引目光，不刺眼
- 阴影范围：深卡片使用 `0 20px 60px rgba(0,0,0,0.4)`，保持沉浸感
- 背景渐变：不应该显得单调，但也不能过度装饰

### 7.2 文字对比度

- 正文 vs 背景：`#b0ada6` vs `#0b1026` ≥ 4.5:1（WCAG AA）
- 琥珀金 vs 深背景：`#d4a853` vs `#07080f` ≥ 3:1（可接受）

### 7.3 可触及性

- 按钮最小点击面积: 48x48px（Mobile）
- 焦点态必须有视觉反馈（border 或 glow）
- 不能只靠颜色区分状态（必须有形状/文字变化）

## 附录：设计决策记录

### A. 为什么选择琥珀金 `#d4a853`

- 在深黑背景上自然发光，不需要额外 glow 也能吸引目光
- 色温偏暖，呼应"暗线"的温度感
- 在节点、按钮、连线中都能统一应用，建立视觉语言一致性
- 对标 Nautilus 杂志的金色印刷感

### B. 为什么是"星网"背景而不是纯色

- 暗示"节点间的暗线连接"，与产品概念一致
- 持续缓动的动画给了页面"呼吸感"，而不是死板
- 星点数量适中（36 个），不会显得复杂或干扰内容

### C. 为什么是单次点击揭示，而不是自动播放

- 尊重用户的思考节奏，有时间消化每一个转折
- 提高了参与感："我在点亮这条路"而不是"我在看视频"
- 符合编辑精选的逻辑：不是全部推送，而是等用户准备好再给下一个

### D. 为什么路线元信息卡要包含"意外指数"

- 给了用户对路径质量的直观认知
- 鼓励用户尝试不同路线：7分 vs 9分有显著区别
- AI 生成时的质量评估有了量化反馈

## 附录：已实现但文档未覆盖的功能

以下功能已在代码中完整实现，文档尚未覆盖，供参考：

- **"先看示例路径"按钮**：Hero 区 ghost 按钮（带 ✦ 图标和脉冲动画），点击直接加载预置 `previewResult` 体验完整流程
- **结果质量提示**：检测到通用桥接词或低惊喜度路径时，状态栏显示提示，可点击"换一条更妙的路径"重试
- **缓存命中提示**：命中 localStorage 缓存时显示"已直接打开你最近一次生成过的路径"
- **"收起重看"按钮**：揭示过程中可重置路径到初始状态
- **路径 Tab 切换**（`path-tabs`）：多路径时顶部标签页切换
- **Hero 装饰 Canvas**：桌面端（≥900px）左侧品牌区下方的 3D 行星轨道动画（`.hero-deco-canvas`）
- **下载分享卡片**：收尾区域 Canvas 生成 1200×630 PNG 并下载
