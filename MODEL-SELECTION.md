# Entanglement 项目模型选型评估报告

> 评估日期：2026-06-16  
> 场景：项目部署在 Vercel 公网（美东节点），Friday 企业内网 API 不可达，需要选一个**公网可用的单一模型**  
> 数据来源：官方定价页、Artificial Analysis Leaderboard、LiveBench-2026-01-08、MiniMax 开放平台

---

## 一、背景与约束

### 为什么 Friday 在公网不能用

本地开发时用的是 Friday（美团企业 API 广场），`FRIDAY_BASE_URL=https://aigc.sankuai.com/v1/openai/native`，这是美团内网地址。

**Vercel 服务器在美国公网，无法访问内网地址。** 所以只要代码部署到线上，`FRIDAY_APP_ID` 这个 provider 就自动失效，需要一个公网可用的 API 替代。

### 选型前提

- 只选**一个**公网可用的模型 provider，不做多模型切换
- 接入方式：在 `api/_lib/llm.ts` 中，`FRIDAY_APP_ID` 失效后自动走 `OPENAI_API_KEY`，**只需在 Vercel 配置 `OPENAI_API_KEY` 和 `OPENAI_MODEL` 两个环境变量即可**，不需要改代码
- 不考虑 GLM-4-Flash：已在项目中明确验证不可用，跨境超时严重，路径质量差

## 二、项目对模型的真实需求

选型要先说清楚任务本质，不然看 benchmark 数字没有意义。

### 任务类型

给定两个毫不相关的词，生成 1-3 条「硬骨架现实路径」，每条路径包含 3-5 个节点，以及大量文案字段（hook / detail / summary 等），以**严格嵌套 JSON** 输出。

这是**深度因果推理 + 中文长文本创意写作 + 结构化 JSON 输出**的三合一任务，对模型要求不低。

### 关键需求

| 需求 | 权重 | 说明 |
|------|------|------|
| 深度推理/因果链路能力 | ★★★★★ | 核心，决定路径是「硬骨架」还是「浅联想」，也决定能否通过 `validateResult()` 质量过滤 |
| 中文写作质量 | ★★★★★ | hook/detail/summary 要有信息密度，不能是通顺废话 |
| 延迟与稳定性 | ★★★★☆ | Vercel `maxDuration: 60s`，目标首次响应 30s 内，超时走服务端多轮重试（`ATTEMPT_TIMEOUTS_MS = [55, 50, 50]s`） |
| JSON 输出合规 | ★★★☆☆ | 不能带 markdown 包裹，否则增加重试率；有 `extractJson()` 兜底但会消耗 token |
| 成本 | ★★★☆☆ | 个人产品，质量优先，但不能离谱 |

### Token 规模（单次请求）

基于 `api/_lib/prompt/entangle.ts` 实际内容测算：

| 部分 | Token 量 | 说明 |
|------|---------|------|
| System prompt（规则 + 6 种妙感类型） | ~1,500 | 规则文本部分 |
| System prompt（9 个完整好例 JSON） | ~3,000 | **成本大头**，每个好例约 330 tokens |
| System prompt（4 个坏例 + 解析） | ~500 | 包含坏例 JSON 和"为什么坏"说明 |
| User prompt（起点/终点 + 本轮要求） | ~150–200 | 词对本身几乎不影响成本 |
| 模型输出（1-3 条路径 JSON） | ~800–1,500 | 2 条路径约 1,000 tokens |
| **单次正常消耗** | **~5,800–6,700** | 正常第 1 轮 |
| 极端情况（3 轮 attempt 全走） | **~17,000** | 每轮都完整带 system prompt |

**对应费用参考（GPT-5.4 mini，2026-06 定价）**

| 场景 | Token | 费用 |
|------|-------|------|
| 正常 1 轮 | ~5,800 | ~¥0.030 |
| 极端 3 轮 | ~17,000 | ~¥0.088 |
| 日均 100 次（正常） | ~580,000 | ~¥90/月 |
| 日均 100 次（极端全重试） | ~1,700,000 | ~¥265/月 |

> **降成本提示**：System prompt 里 9 个完整好例占约 3,000 tokens，是单次消耗的最大来源。如果未来想降成本，精简好例数量（例如保留 5 个）是最直接的方向，理论上可压缩 20–30% 成本，但需实测是否影响生成质量。

## 四、各模型逐项分析

### GPT-5.4 mini ⭐ 推荐

**能力：** LiveBench 67.54，推理 72.50，是 OpenAI 新一代 mini 旗舰，全面超越上一代 gpt-4.1-mini 和 gpt-4o-mini。  
**中文：** OpenAI 系列对中文的理解和写作质量稳定可靠，满足本项目需求。  
**延迟：** TTFT ~10s，输出 180 tokens/s，速度快，安全。  
**成本：** $0.75/$4.50，单次 ~¥0.036，日均 100 次约 **¥108/月**。  
**接入：** 零改动，只改环境变量。  
**结论：** OpenAI 体系内的最优选，比 gpt-4.1 更新更便宜。

---

### Gemini 3.5 Flash ⚠️ 有致命延迟风险

**能力：** LiveBench 75.02，推理 82.00，语言 84.58——**语言分是所有候选中最高的**，综合能力强。  
**中文：** Google 对中文语言能力持续投入，文案流畅度好，但对**中国本土场景**（如「深海采矿→韩国大选」「外卖骑手→自动驾驶」这类需要理解国内制度和产业结构的链路）理解深度不及国产模型。本项目有大量此类场景。  
**延迟：** TTFT 约 18.5s，在当前 60s timeout（`vercel.json maxDuration: 60`）下有充足缓冲，Artificial Analysis 实测也在合理范围内。但真正的风险是 Vercel 公网节点（美东/新加坡）访问 Google API 的跨境网络不稳定——这与 DeepSeek 直连的跨境问题类似，需要通过中转解决。  
**成本：** $1.31/$1.31，单次 ~¥0.035，与 GPT-5.4 mini 相当。  
**接入：** 需改约 15 行代码，注册 Google AI Studio 账号。  
**结论：** 能力优秀，TTFT 在 60s timeout 下安全。**不推荐**——核心障碍是跨境网络稳定性，除非通过 Google 专线或中转站解决。

---

### MiniMax-M3 🔶 活动期间可选，有折扣时间窗口

**能力：** LiveBench 全局 70.02，推理 74.48，语言 76.84，在同价位国产模型里综合表现不错。  
**中文：** MiniMax 深耕中文，对本土场景理解好，文案质量有保证，非常适合本项目对「硬骨架现实链路」的生成需求。  
**延迟：** TTFT ~3.7s，输出 59 tokens/s，速度中等，完全安全。  
**成本：** 当前官网标注「永久五折」，折后 ¥2.10/¥8.40，单次 ~¥0.032，日均 100 次约 **¥96/月**——比 GPT-5.4 mini 略便宜。**但折扣活动随时可能结束**，原价 ¥4.20/¥16.80 约为折后 2 倍，届时性价比下降明显。  
**JSON 合规：** 兼容 OpenAI 接口格式，支持 JSON mode，稳定性良好。  
**接入：** 需改约 15 行代码，注册 MiniMax 开放平台账号（国内平台，注册简单）。  
**结论：** 能力和成本都有竞争力，中文场景表现好。**如果愿意改 15 行代码且折扣期间使用，是 GPT-5.4 mini 的有力替代**。折扣结束后需重新评估性价比。

---

### DeepSeek V4 Flash / V4 Pro 💰 极致性价比

**能力：** V4 Flash（67.25）与 GPT-5.4 mini 持平；V4 Pro（73.58）接近 Gemini 3.5 Flash，推理 82.69，明显更强。  
**中文：** 国内顶级，因果推理和现实机制理解深度出色，非常契合本项目任务。  
**延迟：** TTFT 1.4–1.9s，是所有候选中最低，完全不存在超时风险。  
**成本：** Flash 单次 ~¥0.004，Pro 单次 ~¥0.013，日均 100 次月费仅 **¥12–¥39**，是 GPT-5.4 mini 的 1/9 到 1/3。  
**接入：** 需改约 15 行代码，注册 DeepSeek 账号。  
**结论：** 性价比最优，中文能力强，延迟最好。**如果愿意改 15 行代码，这是长期最优解**，尤其 V4 Pro。

---

### GPT-5.4（旗舰）🏆 能力天花板，暂时溢价

**能力：** LiveBench 80.28，推理 88.12，当前 OpenAI 实际最强。路径质量上限最高。  
**成本：** 单次 ~¥0.18，日均 100 次月费 ~¥540，是 GPT-5.4 mini 的 5 倍。  
**结论：** 现阶段个人产品不需要，等 prompt 调优、路径质量真的到天花板时再升。

## 五点五、当前生产环境实际配置

> 以下为截至文档更新时的线上实际配置，供参考。

当前 [entanglement.doudou.design](https://entanglement.doudou.design) 使用**硅基流动（SiliconFlow）中转站**路由到 DeepSeek V3，通过 OpenAI provider 接入，零代码改动：

```
OPENAI_API_KEY  = <硅基流动的 Key>
OPENAI_BASE_URL = https://api.siliconflow.cn/v1
OPENAI_MODEL    = deepseek-ai/DeepSeek-V3
```

选择这个方案的原因：
- DeepSeek V4 Pro 能力强（推理 82.69）、成本低（日均 100 次月费 ~¥39）
- 硅基流动服务器在亚洲，对 Vercel 节点跨境延迟更友好
- 利用现有 OpenAI provider 路由，只需配置 3 个环境变量，不改代码

## 六、最终推荐与操作

### ✅ 零改动方案（推荐）：GPT-5.4 mini

**在 Vercel 控制台做两件事：**

1. 确认 `OPENAI_API_KEY` 已配置
2. 将 `OPENAI_MODEL` 设置为 `gpt-5.4-mini`

> ⚠️ 代码默认值 `DEFAULT_OPENAI_MODEL` 是旧版 `gpt-4o-mini`，**必须显式设 `OPENAI_MODEL`，否则线上跑的是更弱的旧模型。**
